# Releases

=== "<span style='color: #6d82f6;'>:octicons-tag-24: 0.0.6 - 07 Jun 2026</span>"

<br>

### :material-cash-multiple: Cost Optimization & Efficiency
---

This release introduces multiple backend optimizations designed to reduce operational costs while maintaining data quality and processing performance.

- Reduced infrastructure and API operating costs
- Eliminated unnecessary requests during large-scale data collection  
- Improved request efficiency to maximize throughput with fewer external API calls  
- Reduced infrastructure overhead during high-volume processing operations  

<br>

### :material-rocket-launch-outline: Reliability & Processing Improvements
---

Focused on improving execution reliability, fault tolerance, and long-running job stability.

- Improved background processing resilience across concurrent workloads  
- Added safeguards to prevent stalled or unresponsive processing tasks  
- Enhanced handling of partial failures so successful operations continue uninterrupted  
- Improved stability when processing large datasets and extended scraping sessions  
- Strengthened overall execution consistency across production environments  

<br>

### :material-email-outline: Communication Infrastructure
---

Enhanced outbound communication reliability through improvements to the email delivery stack.

- Migrated to a new email delivery provider for improved message reliability  
- Improved email infrastructure stability and delivery consistency  
- Enhanced support for production-scale outbound communications  

<br>

### :material-chart-line: Monitoring & Visibility
---

Improved system observability to provide better operational awareness and troubleshooting capabilities.

- Enhanced warning and alert visibility across processing pipelines  
- Improved monitoring of execution states and background operations  
- Strengthened logging consistency for easier diagnostics and issue tracking  

<br>

### :material-cogs: Platform Stability & Code Quality
---

Additional improvements focused on maintainability, scalability, and long-term platform reliability.

- Simplified internal processing workflows for improved maintainability  
- Refined execution flow to improve scalability and future extensibility  
- Improved internal data handling consistency across processing layers  
- Cleaned up code structure and naming conventions for easier maintenance  
- Resolved multiple edge cases that could interrupt processing workflows
